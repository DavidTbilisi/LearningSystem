# Onboarding Path — Learn the Mnemonic Stack by Levels

This file is the **meta-manual**: how to install the ecosystem in your head **without drowning**. It sits beside `SKILL.md` (what exists) and answers **in what order**, **how deep each pass goes**, **how cards age**, and **when to delete work** that no longer earns its keep.

Use it in three passes: pick **minimal** until reviews feel boringly easy, then **mid**, then **maximal** only if you are encoding graphs, large numeric spaces, or multiple domains in parallel.

---

## Preconditions (every level)

Before any path, commit to four invariants:

1. **One scheduler** — Anki (or equivalent) is the clock; palaces and CAST are the geography. Do not maintain two competing interval systems for the same fact.
2. **One session log** — after each study day, one line: *systems touched · new cards · reviews due tomorrow · felt load (low/med/high)*. Measurement reads this later.
3. **Rewrite beats grind** — if a card fails twice after a conscious imagery fix, it is a **content bug**, not a discipline bug. See pruning rules.
4. **Comprehension before CAST on greenfield topics** — do not graph what you do not yet understand at sentence level. Run at least **Locate + Represent** from the comprehension protocol before locking edges.

---

## Level A — Minimal path (first operational loop)

**Goal:** a believable end-to-end loop — *encode → schedule → review → one repair move* — with **two** mnemonic substrates only.

**Time horizon:** about **two to four weeks** to first plateau (reviews predictable, no daily panic).

### Week 1 — Skeleton

- Read `SKILL.md` once for the map; skim comprehension protocol gates (no mastery yet).
- **Numeric:** install **Major 00–99** *or* a **20-image bootstrap** (00–19) if full hundred feels heavy; pick one and finish it before expanding.
- **Operational:** choose **one** small palace (8–12 loci). Name it; never change the path order this month.
- **Anki:** cap **10 new cards/day**; tag every card with `#system` and `#domain`.
- **Retrieval doc:** read Component 1 (templates) only; copy **one** template (Major or NEDF stub) into a note you reuse.

### Week 2 — Close the loop

- Add **review-only** palace walk: 5 minutes, same path, recall only what you already placed (no new loci).
- If any card lapses **three times**, apply **one** weak-link repair from retrieval protocol (simplify image, split card, or add prerequisite mini-card) — **before** it becomes a leech.
- **Stop adding** new subsystems this week unless reviews are under 30 minutes total.

### Exit criteria for leaving Minimal

You can promote to **Mid** when **all** are true for seven consecutive days:

- Reviews finish within your chosen daily ceiling without skipping “hard” on everything.
- You can verbalize **when** you would pick Major vs palace (even if clumsily).
- Zero cards above **8 lapses** without a rewrite ticket logged.

## Level B — Mid path (core encoding identity)

**Goal:** stable **concept encoding**, **chunked numbers**, and **one relational preview** — enough to learn real material in a job or degree.

**Time horizon:** about **eight to twelve weeks** from end of Minimal.

### Weeks 3–4 — Concepts and chunks

- **NEDF:** encode **five** real concepts from your domain with full four-slot scenes; add cloze variants only after forward recall works.
- **SEM3 + Major** *or* **PAO** for 3–4 digit material: pick **one** chunking style and hold it for the month (do not parallel both until reviews stay flat).
- Raise new cards to **20/day** only if weekly review time stayed flat the prior week.

### Weeks 5–6 — Binary / hex and CAST preview

- **Binary-hex:** learn the **4×4 elemental matrix** cold; drill **16** one-hex cards to sub-three-second recall.
- **CAST:** encode **exactly three** real edges from one project (code, math, or argument graph). Georgian nodes optional — use placeholder animals if needed, but edges must be honest.
- **Palace:** second route or second room chain for a different domain; still **one walk order** per palace.

### Weeks 7–8 — Retrieval and collisions awareness

- Read retrieval protocol **Components 2–3** (cadence + repair). Schedule **one** weekly full palace walk.
- Skim **collisions** doc; tag any card that shares an image with another system with `#collision-risk`.

### Exit criteria for leaving Mid

- You have **at least one** living CAST mini-graph you still use when debugging or studying.
- Numeric drills for your chosen chunk size are **automatic** (no arithmetic on the card).
- You have used **measurement framework** once to label your weakest dimension honestly.

## Level C — Maximal path (full stack ownership)

**Goal:** **Georgian + CAST** at scale, **procedure chains** (SPEAR), **domain pattern** hooks, and **measurement-driven** belt progression — without hero days you cannot repeat.

**Time horizon:** **six months plus**, iterated in **four-week blocks**. Each block adds **one** new subsystem or **one** new domain — never both at once.

### Block themes (repeat in order)

1. **Graph month** — expand CAST; refine Georgian disambiguation; one collision audit.
2. **Procedure month** — SPEAR one real algorithm or workflow; link to palace corridors.
3. **Pattern month** — add **five** domain patterns you actually hit at work; encode triggers only.
4. **Meta month** — metacognitive checklist weekly; measurement LPQ; prune deck and palaces per rules below.

### Load caps (Maximal)

- **New cards:** start at **30/day**; raise toward **40+** only if **retention seven-day rolling ≥ 85%** and session time is stable.
- **Review ceiling:** hard-stop daily reviews at **400**; overflow → reschedule **tomorrow** as maintenance, not as a guilt stack. If you hit the ceiling often, you are above capacity — drop new cards first.

### Exit criteria (there is no “done”)

Maximal is **sustained**: you keep measurement + quarterly collision passes + one palace retirement cycle per year. The stack is “complete” when maintenance is boring.

## Spacing defaults (Anki-aligned)

These are **defaults**, not laws. Tune after you have **four weeks** of true retention data. Prefer **FSRS** (Anki 23+) when available; the numbers below assume classic SM-2 style if you must fall back.

### Scheduler

| Setting | Default | Note |
|---------|---------|------|
| Algorithm | **FSRS** with personal stability | Reschedule after ~200 graded reviews for fit |
| Learning steps | **1m · 10m · 1d** | Third step is first “sleep” bridge |
| Graduating interval | **1 day** | Keeps young cards near each other |
| Starting ease | **2.50** | Lower if you feel punished on “Good” |
| Easy bonus | **1.30** | Rarely use Easy on young cards |
| Interval modifier | **1.00** | Drop to **0.85** during high life load |
| Maximum interval | **36500 days** | Effectively “never cap” for core facts |

### New cards and review caps by level

| Level | New/day (soft cap) | Suggested review ceiling | Leech threshold (lapses) |
|-------|---------------------|----------------------------|---------------------------|
| Minimal | 10 | 150 | 8 |
| Mid | 20 | 280 | 8 |
| Maximal | 30–40 | 400 | 8 |

### Interleaving rule

- **Same system first:** when possible, finish reviews for one tag (`#cast`) before opening another (`#major`) in the same session — reduces context thrash. If Anki mixes anyway, accept it; do not fight the tool mid-session.

### Young vs mature

- Cards with interval **under 21 days** are “young” — protect them from bulk suspends.
- Cards with interval **over 90 days** are “mature” — safe to suspend entire tags when a project ends (see pruning).

## Pruning rules (cards, palaces, graphs)

Pruning is **quality control**, not defeat.

### Anki cards — delete when

1. **Superseded** — a newer card encodes the same fact with strictly easier retrieval.
2. **Project dead** — the codebase or course unit shipped and you will not touch that domain for **twelve months**.
3. **Leech** — **≥ 8 lapses** after one rewrite attempt → **suspend**, then **delete or replace** within a week. Never let leeches idle in suspended purgatory.

### Anki cards — merge when

1. Two cards are **forward–reverse of the same edge** and you always fail them together — merge into one bidirectional with two prompts on the back.
2. **Micro-cards** that only exist for a single exam list — merge into a cloze list card after the exam if you still need the bundle.

### Palace loci — retire when

1. The spatial story **no longer matches** the real topology (refactor moved the graph) — **migrate** once, then erase old props deliberately.
2. A locus has been **empty six months** — strip decorations; reuse the slot in a new block theme.

### CAST / Georgian graph edges — collapse when

1. Two nodes **merged in reality** (classes collapsed) — keep one node card; delete redundant edge cards.
2. An edge was **hypothesis** and falsified — delete the edge card and add a **one-line “was wrong”** card if you need the mistake as a lesson (tag `#falsify`).

### Deck-level hygiene (monthly, 15 minutes)

- Sort by **lapses**; top **20** get a rewrite or delete decision.
- Sort by **interval**; bottom **50** mature cards in dead tags → suspend batch.

## Cadence quick reference

| Cadence | Action |
|---------|--------|
| Daily | Anki reviews to zero or to ceiling; one-line session log |
| Weekly | One palace walk **or** one CAST path narration; check leech queue |
| Monthly | Prune pass; collision tag audit; adjust interval modifier if load was brutal |
| Quarterly | Measurement LPQ; pick next four-week block theme (Maximal) |
| Yearly | Retire one palace or major deck branch; archive tags for ended domains |

---

## Under-load policy (stress, sleep, illness)

If **sleep under six hours average** or acute stress: **freeze new cards** for **48h minimum**; interval modifier **0.80**; only reviews + **one** repair block (≤15 min). Palaces are read-only — no new loci. This is **maintenance mode**, not regression.

---

## Verification cadence (fidelity to reality)

For every **active** domain (job stack, degree unit, personal project): **one hour per month** reading source truth and **spot-checking five random cards** + **two random CAST edges**. Failed checks → `#needs-rewrite` tag and scheduled fix before new encodings in that domain.

## Promotion checklist (summary)

| Move | Requirement |
|------|----------------|
| Minimal → Mid | Stable daily reviews; Major or bootstrap complete; one palace trusted |
| Mid → Maximal | NEDF + one numeric chunk style automatic; three honest CAST edges alive; retrieval cadence adopted |
| Stay in level longer | Measurement says one dimension is red **or** life load is high — repeat block |
