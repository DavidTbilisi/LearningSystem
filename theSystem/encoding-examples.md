# Encoding Examples — Worked Walkthroughs

These examples show exactly how to stack systems for real data.

---

## Example 1: 4-digit number → SEM3 + Major

**Number: 2743**

1. Split: `27` (SEM3 prefix) + `43` (Major suffix)
2. SEM3 `27` → category 2 (Smell), item 7 → **Coffee**
3. Major `43` → **Ram**
4. Scene: *A Ram charging through a coffee shop, coffee smell everywhere*
5. The smell category (SEM3) makes the scene multi-sensory — you don't just see it, you smell it

---

## Example 2: 8-digit number → two SEM3+Major chunks

**Number: 27436518**

1. Split into two 4-digit chunks: `2743` + `6518`
2. Chunk 1 `2743`: SEM3 `27`=Coffee + Major `43`=Ram → *Ram in coffee shop*
3. Chunk 2 `6518`: SEM3 `65`=Animals→Elephant + Major `18`=Diva → *Elephant acting like a Diva*
4. Place image 1 at locus 1, image 2 at locus 2 in your mind palace
5. Walk the palace to recall: Coffee Ram → Diva Elephant → 27436518

## Example 3: 10-digit number → SEM3+Major + SEM3+Major + Peg Matrix

**Number: 2743651892**

1. Split: `2743` + `6518` + `92`
2. Chunks 1 & 2: as above
3. Remainder `92` → Peg Matrix: tens=9 (Wine) + units=2 (Swan) → *Wine + Swan*
4. Three loci in your palace: Coffee Ram → Diva Elephant → Wine Swan

## Example 4: Phone number (3+3+3) → SEM3 per group

**Georgian number: 598 444 444**

Option A — SEM3 for each 3-digit group:
- `598` → SEM3 `5`=Sensation, item `98`=? (use nearest: 9×10+8 → category 5, entry 98 ≈ Peace area) → Flying + Beef
- `444` → SEM3 `4`=Touch, item `44`=Rock + Major `44`=Ra-ra
- `444` → same image repeated = easy!

Option B — PAO structure (Person + Action + Object per 3-digit group):
- Each 3-digit group encodes a mini-story
- Useful when numbers repeat (444 444 → same PAO = easy to recall)

## Example 6: Mind Palace as domain prefix

**Memorizing CS concepts at university**

- Palace = your actual university building
- Entrance hall → concept 1 (e.g. algorithm complexity: SEM3 `05`=Lightning + Major `32`=Man → Lightning Man)
- Lecture hall → concept 2
- Each locus inherits the "CS / university" context from the palace
- When you mentally walk into that building, the domain primes your recall

## Design Example: New system for Georgian phone numbers

**Problem**: 9-digit numbers in format 5XX XXX XXX

**Approach**:
1. First digit is always 5 → ignore or use as palace anchor ("5 = phone context")
2. Remaining 8 digits → two SEM3+Major chunks: `XX` + `XX` + `XX` + `XX`
3. Or: treat as 3+3+2 → SEM3 group + SEM3 group + Major remainder
4. Build a PAO system where: hundreds digit = Person, tens = Action, units = Object
5. This gives one vivid scene per 3 digits → 3 scenes for 9 digits → very manageable

## Example 7: Georgian phone number (9 digits) → PAO × 3

**Number: 598 444 444**

1. Split: `598` + `444` + `444`
2. `598` → Person 5 (Loki) + Action 9 (Burning) + Object 8 (Flag) = *Loki burning a flag*
3. `444` → Person 4 (Robin Hood) + Action 4 (Ripping) + Object 4 (Ring) = *Robin Hood ripping a ring*
4. `444` → same scene repeated = *Robin Hood ripping a ring*
5. Three loci in palace: Loki burning flag → Robin Hood ripping ring → Robin Hood ripping ring
6. The repeated scene for `444` is a gift — identical story twice is impossible to forget
