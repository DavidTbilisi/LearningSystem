# Georgian Alphabet System — 33 Canonical Entries

The full 33-letter Georgian mnemonic system. Each letter has **4 slots**, all
starting with the same Georgian letter sound:

1. **Animal** — node identity (also the month's totem)
2. **Person** — human name (for PAO-style scenes and disambiguation)
3. **Adjective** — state/condition modifier
4. **Environment** — location/setting

This single table powers three things simultaneously:
- **CAST node encoding** (Animal = identity, Environment = cluster, Adjective = state)
- **Calendar system** (33-slot extended calendar — one letter per month/week)
- **General-purpose peg** (when you need 33 distinct imagery slots)

## How the Four Slots Compose

Every full scene = **Animal + Person + Adjective + Environment**, all sharing the same letter sound. Example for ა:

> *An exalted eagle (არწივი) named Anano (ანანო) stands ამაღლებულ (exalted) აივანზე (on a balcony).*

The alliteration locks the letter in. The four elements give four retrieval paths:
hear the animal → recall the person; hear the adjective → recall the location.

## Usage Modes

### Mode A — Calendar (33 months / weeks / units)

The 33-entry structure maps to an extended calendar. Each month is a full
mnemonic sentence. For dates within a month, append a Major System image:

- **Month + day:** Month-scene + Major image for the day
  Example: ა (January) + day 17 (Dog) → *Anano's eagle holds a dog on the exalted balcony*
- **Full date YYYY-MM-DD:** SEM3+Major for year + Georgian letter for month + Major for day

### Mode B — CAST node encoding

Pick the Georgian letter whose animal matches the node's domain meaning:

- **Animal** = node identity
- **Environment** = cluster (or yields to palace location if palace is present)
- **Adjective** = state/condition
- **Person** = (optional) disambiguation when multiple instances of same node type

Combined with CAST edges (see `cast-system.md`), this gives full graph encoding.

### Mode C — 33-slot peg system

When you need an ordered list of 33 items, use the letter sequence. Each slot
offers four hooks (animal, person, adjective, environment), so you can layer
multiple items per slot by attaching different items to different hooks.

### Mode D — Disambiguation via Person slot

In CAST networks, if you have two Routers (both Eagles), use the Person slot
to disambiguate: Router-A = "Anano's eagle," Router-B = "a different eagle
named [person2]." The Person slot is reserved for this.

## Multi-Animal Letters

Some letters have multiple canonical animals (ვ, კ, ო, ჭ). Pick based on context:

- **ვ:** Whale (ვეშაპი) for large/oceanic nodes, Tiger (ვეფხვი) for predatory/aggressive
- **კ:** Cat (კატა) for small/agile, Rabbit (კურდღელი) for fast/reactive, Kitten (კნუტი) for new/small instances
- **ო:** Hoopoe (ოფოფი) for flying/messenger nodes, Spider (ობობა) for web/network nodes
- **ჭ:** Ladybug (ჭიამაია) for small/helpful, colorful bug for variegated/mixed

Document your choice per domain so the same node isn't encoded two ways.

## Integration Notes

- **Same letter across domains:** A "Whale" in a codebase palace and a "Whale"
  in a biology palace are different animals in different palaces. The palace
  context disambiguates — no collision.
- **Georgian + Major:** The Person slot aligns with Major System consonants
  where possible (ბექა starts with B=9, etc.), so you can encode numbers by
  choosing the letter carefully.
- **Georgian + SEM3:** Don't combine them in one scene — they compete for
  the scene's "character slot." Use Georgian for graph work, SEM3 for
  numeric work.
