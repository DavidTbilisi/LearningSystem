# Metacognitive Checklist

Elite learners do something most learners don't: they **watch themselves
learn**. They notice when they're fake-reading, when they're confabulating,
when they've drifted from learning to performing. This is the meta-level
audit.

**The principle:** object-level work (the content) doesn't improve reliably
unless the meta-level (the quality of the process) is monitored. The object
level is what you're learning; the meta level is *how well* you're learning it.

This file is a set of **self-audit questions** to run at specific checkpoints
during learning and problem-solving.

---

## The Four Checkpoints

Every session has four moments where metacognitive audits pay the highest dividends:

1. **Before starting** — orient: what's the actual goal?
2. **Mid-session** — check: is the process working?
3. **End of session** — review: what did I actually retain?
4. **Next day** — test: does it still hold?

Different questions for each checkpoint.

---

## Checkpoint 1 — Before Starting

Run these in under 60 seconds before opening any source material.

- **What am I trying to learn?** (One sentence. If vague, refine.)
- **Why?** (The application or decision that depends on this knowledge.)
- **How will I know I've learned it?** (Concrete test — can explain, can solve, can apply.)
- **What's my time budget?** (Bounded attention prevents drift.)
- **What's the minimum viable outcome?** (If I get interrupted halfway, what do I still want?)
- **What's one thing I shouldn't try to learn right now?** (Scope discipline.)

**Red flags:**
- Can't state the goal → you're drifting into content without purpose
- Can't state the test → you won't know if you succeeded
- "I'll know when I know" → self-deception

---

## Checkpoint 2 — Mid-Session (every ~25 min)

Pause briefly and run these. This is the highest-yield checkpoint because
it catches drift before it becomes a wasted session.

### The Four Questions of In-Progress Learning

1. **Am I actually learning, or am I scanning?**
   - Test: can I summarize the last 10 minutes in 3 bullets?
   - If no → you were fake-reading. Go back.

2. **Am I confabulating?** (Filling gaps with plausible-sounding guesses)
   - Test: for each claim I'd make right now, can I point to the source?
   - If no → you've built on unverified foundations. Mark those claims.

3. **Is this above my level?**
   - Test: how many unfamiliar terms per page? 0–1 is fine, 2–3 is stretching, 4+ means the prerequisite stack isn't there.
   - If above → drop a level (confusion-triage section A).

4. **Am I performing understanding, or having it?**
   - Test: if I had to teach this right now, to someone who'd ask follow-ups, could I?
   - If no → you have recognition, not comprehension.

### Signal: the "reading speed" check
- If I'm reading faster than usual → probably scanning
- If I'm reading slower than usual and mind is drifting → the material is above my level or I'm tired
- If I'm reading slower than usual and absorbing → good state, continue

---

## Checkpoint 3 — End of Session

Run before closing the material. ~5 minutes.

### The Regeneration Pass
- Close the source.
- On a blank page, write what I now know about [topic].
- Compare to what I set out to learn.
- **Gap analysis:** what's missing? That's what I actually need to revisit.

### The Confabulation Pass
- Of what I wrote, what am I *certain* about vs. *assuming*?
- Mark the assumptions. Next session → verify them.

### The Encoding Decision
- Which items from today should become mnemonic encodings?
- Which should just be Anki facts?
- Which should be skipped (low-value)?

### The Next-Action Note
- What's the single next thing I should do with this learning?
  - Apply it to a real problem within 48h?
  - Read the next chapter?
  - Backfill a prerequisite I noticed was shaky?
- Write it down. Future-you will thank present-you.

---

## Checkpoint 4 — Next Day

Before reviewing yesterday's material in Anki or notes:

- **Cold-recall test:** what can I still regenerate, 24 hours later, without looking?
- **What decayed fastest?** That's what needs stronger encoding.
- **What still feels solid?** That's what's actually retained.

Anki reviews will catch the atom-level decay. This check catches the
structural decay (the big picture) that Anki can't test.

## The 9 Common Self-Deception Patterns

Elite metacognition means recognizing these in yourself. Each is a specific
failure mode with a specific name. Name it when it's happening and the
spell breaks.

### 1. The Illusion of Explanatory Depth
**Symptom:** "Yeah I understand how bikes work." Asked to draw the chain path → draws it wrong.
**The trick:** you confuse *familiarity with the word* for *knowledge of the mechanism*.
**Detection:** Gate 5 of the Comprehension Protocol — can you regenerate it?

### 2. The Fluency Trap
**Symptom:** the explanation flows smoothly → feels like understanding.
**The trick:** your brain rewards smooth parsing with a confidence signal, even if the content didn't stick.
**Detection:** close the book and try to reproduce. Fluency disappears fast.

### 3. The "I Already Know This" Reflex
**Symptom:** encountering familiar-sounding content, skipping past it.
**The trick:** pattern-matching on surface features. The content may be deeper than the surface.
**Detection:** if you think "I already know this" → stop and test it. If you pass, fine. If you don't, you just dodged a hole.

### 4. The Performance Substitution
**Symptom:** you're explaining things to yourself, sound smart, feel productive — but haven't tested anything.
**The trick:** mistaking the *feeling of capability* for capability.
**Detection:** translate explanation into a problem. Solve it. If you can't, the performance was hollow.

### 5. The "One More Book" Delay
**Symptom:** before attempting the thing, you want to read one more source.
**The trick:** consumption feels safer than production; you're avoiding the test.
**Detection:** ask "would attempting and failing teach me more than another source?" Usually yes.

### 6. The Jargon Shield
**Symptom:** using technical terms to *cover* the gap where explanation should be.
**The trick:** jargon passes as content to other jargon-users, including yourself.
**Detection:** can you explain without the technical term? If no, you're using it to hide.

### 7. The Confidence-Evidence Inversion
**Symptom:** your confidence in something is *higher* the less you've tested it.
**The trick:** fresh claims feel strong because no counterexamples have surfaced yet.
**Detection:** for any high-confidence belief, ask "when was this last tested against reality?"

### 8. The Goal Drift
**Symptom:** you started wanting to learn X; two hours later you're down a rabbit hole on Y.
**The trick:** each step felt locally reasonable; the trajectory is still wrong.
**Detection:** Checkpoint 2 Question 1 — am I still on the original goal?

### 9. The Fake-Application
**Symptom:** "I applied it" actually means "I ran through an example in my head."
**The trick:** mental simulation is not the same as real-world application. Reality has friction imagination doesn't.
**Detection:** did *real external feedback* occur (code ran, problem solved, claim tested against evidence)? Or only internal?

## Metacognitive Moves During Problem-Solving

Different from learning — these run during the Heuristic Palace walk.

- **"Am I stuck or am I slow?"** — stuck = no progress despite effort; slow = progress but below pace. Different responses.
- **"What would change my answer?"** — if nothing would, you're attached to the current solution, not reasoning.
- **"What's my confidence that this approach will work?"** — if < 30% and you've spent > 30% of budget, switch.
- **"Am I optimizing for solving or for not-being-wrong?"** — the latter is social-posture, not reasoning.
- **"What would this look like if it were easy?"** — if it suddenly looks easy from a new angle, you were attacking wrong.

## The Weekly Meta-Review (~10 min, once a week)

Go deeper than daily checks. This is how you tune the meta-level itself.

1. **Which sessions were productive? What do they have in common?**
2. **Which were wasted? What pattern caused it?**
3. **What concept do I *claim* to understand that I probably don't?** (Test it.)
4. **What's one piece of my system (encoding, palace, Anki workflow) that's not paying off? Adjust or drop.**
5. **What's one thing I notice myself avoiding? Why?**

Short entries. One or two sentences per question. Track over time —
patterns emerge by week 4.

---

## The Meta-Trap — Don't Over-Meta

Metacognition has a cost. At some point it becomes procrastination disguised
as rigor. Signs you've gone too meta:

- You've spent > 10% of session time on meta-checks
- You're planning to plan to plan
- You haven't produced object-level content in a week but have refined the system 3 times
- Your Anki reviews are working fine but you keep rebuilding card templates

**Rule:** meta is a *tool*, not a pastime. Use it at checkpoints, then shut up and do object-level work.

---

## Integration with Other Systems

- **Comprehension Protocol** — the object-level procedure. This checklist audits *whether you're following it honestly*.
- **Confusion Triage** — when stuck on understanding. This checklist catches the illusion of understanding *before* triage is needed.
- **Heuristic Palace** — problem-solving moves. This checklist audits whether you're actually using them vs. pretending.
- **Retrieval Protocol** — Anki + palace walks. This checklist audits whether the retrieval is producing real or illusory mastery.

This checklist is **transverse** — it runs above all other systems.

---

## Key Principles

1. **Object-level work without meta-level audit regresses to mediocrity.** Smart people with bad meta-monitoring plateau.
2. **Self-deception is the default state.** Metacognition is the active correction.
3. **Name the failure mode to neutralize it.** "Oh, that's the Jargon Shield firing." Once named, it loses power.
4. **Checkpoints beat continuous monitoring.** Always-on meta is exhausting and counterproductive.
5. **Regeneration is the only honest test.** Everything else is vulnerable to self-deception.
6. **The meta-layer has a budget too.** Don't let rigor become procrastination.
7. **Weekly review compounds.** Pattern recognition about your own learning is the highest-ROI audit.
